# Visual Direction

## Objetivo

O portfólio deve comunicar profundidade de pensamento, visão sistêmica e capacidade de atuar em produtos complexos.

A estética não deve parecer um site corporativo, uma landing page SaaS ou um template de portfólio.

O objetivo é transmitir a sensação de que cada detalhe foi projetado com intenção.

# Linguagem Visual

## Tipografia

A tipografia é o principal elemento visual do projeto.

Ela deve carregar grande parte da hierarquia e da personalidade.

Priorizar:

- títulos grandes
- contraste forte entre níveis
- blocos longos confortáveis para leitura

---

## Espaçamento

Muito espaço negativo.

O conteúdo deve respirar.

O layout deve transmitir confiança para deixar áreas vazias.

---

## Imagens

As imagens não são decoração.

Cada imagem deve servir para:

- comprovar uma decisão
- evidenciar uma descoberta
- mostrar uma transformação

---

## Diagramas

Diagramas devem aparecer como parte da narrativa.

Mostrar:

- sistemas
- relações
- fluxos de decisão
- arquiteturas

Não apenas fluxos de usuário.

---

## Motion

Motion discreto.

O objetivo não é impressionar.

É criar ritmo.

A sensação deve ser próxima de uma leitura guiada.

---

# Estrutura dos Cases

Os cases devem se comportar como reportagens investigativas.

Não como apresentações corporativas.

Cada bloco deve criar curiosidade para o próximo.

Cada evidência deve sustentar uma conclusão.

Cada conclusão deve justificar a próxima decisão.

---

# IA na Interface

A IA não deve ser apresentada como hype.

Ela deve aparecer como infraestrutura.

Quando surgir nos cases, deve ser percebida como parte do sistema e da operação, não como elemento promocional.

---

# Critério de Aprovação

Se removermos todas as imagens e deixarmos apenas títulos, textos e estrutura, o portfólio ainda deve transmitir senioridade, profundidade e clareza.
